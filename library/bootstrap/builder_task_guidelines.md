# Правила постановки задач для antiengine-builder

> [!TIP]
> **antiengine-builder** — это специализированный саб-агент, предназначенный для написания правил, воркфлоу, скриптов и скиллов для пополнения центральной библиотеки.

Когда вы делегируете задачу Билдеру (через `invoke_subagent`), придерживайтесь следующего формата:
1. **Контекст:** Опишите проблему или концепцию максимально подробно. Если вы дадите абстрактную задачу (например, "Сделай скрипт"), Билдер имеет право **отклонить её** и потребовать от вас уточнений.
2. **Тип ресурса:** Четко укажите, что именно должен создать Билдер (например, "Создай правило для работы с TypeScript", "Создай bash-скрипт для деплоя", или "Создай специализированного субагента-ревьюера для Swift").
3. **Ожидания:** Укажите, что он должен изучить перед созданием (например, "Изучи лучшие практики Apple перед написанием").
4. **Требования к скриптам:** Если задача подразумевает создание исполняемого скрипта, обязательно напомните Билдеру, что он должен встроить стандартную систему ежедневного логирования (папка `logs/имя_скрипта/` и файлы `YYYY-MM-DD.log`). Без этого скрипт не пройдет защиту.
5. **Сохранение:** Обязательно напомните ему: "Используй `save_to_library`, чтобы сохранить этот ресурс".

**Пример промпта для Билдера:**
*"Контекст: Мы начинаем работу над проектом Swift. Создай правило (rule) с лучшими практиками архитектуры SwiftUI. Обязательно проведи веб-поиск перед написанием. Если в библиотеке уже есть похожее правило, изучи его и не перезаписывай, а создай новое с расширением. Сохрани готовый markdown-файл в библиотеку используя `save_to_library`."*
